# Project Documentation & Recent Updates

This document outlines the recent changes made to the project, how the map generation works, and important setup instructions for developers.

## Biome System & Map Generation
The map generation is powered by a `BiomeData` resource system, allowing the game to easily swap between different visual themes (like Snow or Grass).

*   **Sequential Biome Rotation:** `MapGenerator.gd` automatically rotates through all the biomes assigned to its `biomes` array. This is tracked using a `static` variable so that every time the map is regenerated, it seamlessly transitions to the next biome in the list rather than picking one at random.
*   **Quick Reloading (The `R` Key):** To quickly test the map generator and see the biome rotation in action, **press the `R` key** while the game is running. This is handled by `CameraController.gd` and will instantly reload the current scene. 
	* *Note:* Restarting the game from the Godot editor wipes the game's memory, which resets the static variable back to `0`. Always use the `R` key in-game when you want to verify the rotation logic!

## Enemy Pathing & Coordinate Alignment
Enemies track waypoints generated by the `Spawner`. 
*   **Local Coordinate Tracking:** Because the Spawner provides local coordinates (relative to the `MapGenerator` node), the `Enemy.gd` script relies exclusively on local `position` rather than `global_position`. This ensures that even if the map itself has a world offset (like a Z-axis shift), the enemies will always stay perfectly aligned and centered on the generated path tracks.
